PROJECT SPEC: Sunshine World Kids Rhyme & Story App – Initial Wireframe Prototype

Goal:
Build an initial clickable/prototype wireframe for a kids’ rhyme and storytelling app called “Sunshine World”. This prototype is UI/UX-focused only. Do NOT implement backend/server communication, authentication, background sync, security, real account login, cloud storage, or SD card writing logic yet. Use local/static media files and mock data.

The prototype should demonstrate:
1. App navigation structure
2. Home screen
3. Rhymes library
4. Stories library
5. Playlist overview in Kid Mode
6. Parent PIN access screen
7. Playlist detail/editor in Parent Mode
8. Player screen with scene-based visual read-along
9. Register Toy screen
10. OTP verification / claim device screen

Use the supplied screenshots as the visual design reference. Use this spec as the UX and screen behavior guide.


==================================================
1. APP NAME / BRANDING
==================================================

App name:
Sunshine World

Brand feeling:
A magical, safe, cheerful children’s audio world for rhymes, stories, playlists, and toy content management.

Visual tone:
- Soft 3D cartoon style
- Rounded glossy UI components
- Pastel colors
- Warm, magical, child-safe
- Parent-friendly but playful
- Storybook landscape backgrounds
- Wooden bottom navigation dock
- Big readable typography
- Minimal visual clutter
- Large touch targets

Mascot:
Toto the Talking Toy

Toto visual description:
- Round pastel blue toy/speaker character
- Cream face
- Big shiny blue eyes
- Rosy cheeks
- Friendly smile
- Small antenna/knob on top
- Circular speaker grille on belly
- Colorful buttons: pink play button, yellow star button, green music note button
- Used as helper/guide character across screens


==================================================
2. GLOBAL NAVIGATION
==================================================

Bottom dock must be present on main app screens.

Bottom dock items, exactly in this order:
1. Home
2. Rhymes
3. Stories
4. Playlists
5. Parents

Icons:
- Home: house icon
- Rhymes: music note icon
- Stories: open book icon
- Playlists: playlist/folder/list icon
- Parents: adult + child icon

Selected tab:
- Highlight with blue glowing rounded tile/background
- Keep other tabs visible but unselected

Removed/Not used in prototype:
- Explore
- Rewards
- Safe Space

Screen routing:
- Home tab -> Home screen
- Rhymes tab -> Rhymes library screen
- Stories tab -> Stories library screen
- Playlists tab -> Playlists overview screen in Kid Mode by default
- Parents tab -> Parent Access PIN screen unless Parent Mode is already unlocked


==================================================
3. APP MODES
==================================================

There are two UI modes:

A. Kid Mode – default
Available actions:
- Browse Rhymes
- Browse Stories
- Play individual items
- Open Playlists
- Play playlists
- Open player

Hidden in Kid Mode:
- Add to Playlist
- Create Playlist
- Rename Playlist
- Delete Playlist
- Remove playlist item
- Reorder playlist item
- Export to SD card
- Toy registration management, except through Parents flow

B. Parent Mode – mock unlocked state
Parent Mode is unlocked after entering mock PIN.

Available actions:
- Add items to playlist
- Rename playlist
- Remove playlist items
- Reorder playlist items
- Export playlist to SD card
- Register/claim toy
- Manage playlist details

Prototype behavior:
- Use a simple mock PIN such as 1234.
- After PIN entry, set local UI state: parentMode = true.
- Parent Mode can be reset by refresh or an “Exit Parent Mode” mock control if added.
- No real authentication.


==================================================
4. DESIGN SYSTEM
==================================================

Overall style:
- Bright fantasy landscape backgrounds
- Sky blue gradients
- Fluffy clouds
- Hills, flowers, rainbow, castle motifs
- Rounded cream/white content cards
- Wooden bottom navigation dock
- Glossy buttons
- Soft shadows
- Large icons
- Large readable labels

Recommended colors:
- Sky blue: #5EC9FF / #74D5FF
- Deep blue: #267EDB
- Sunshine yellow: #FFD34D
- Warm orange: #FFB347
- Pink: #FF6FAE
- Lavender/purple: #8E63E9
- Mint green: #77D96B
- Cream card: #FFF6E8
- Dark readable text: #3B2A20
- Secondary purple text: #6651B8
- Wooden dock brown: #A9662D / #D8944B

Typography:
- Use rounded, kid-friendly font.
- If custom font unavailable, use system fallback:
  - Primary: Nunito, Baloo 2, Quicksand, Fredoka, or system sans-serif
- Titles should be large and bubbly.
- Body text should be clear and readable.

Button style:
- Minimum height: 48px
- Rounded corners: 18–28px
- Soft shadow
- Icon + label where useful
- Use large play buttons for child actions

Cards:
- Rounded corners: 20–30px
- Cream/white background
- Soft drop shadow
- Slight inner highlight/gloss if easy
- Avoid cramped spacing

Touch target:
- Minimum 44px
- Prefer 56px+ for child-facing controls

  id: string,
  title: string,
  description?: string,
  coverImage: string,
  itemIds: string[],

- Maximum playlists: 10

- Hardcode 5 playlists.

- If using localStorage, store mock playlists locally only.

13. SCREEN 7 – PLAYER / READ-ALONG SCREEN
==================================================

Purpose:
Play local audio with scene-changing visuals and live read-along transcript.

This is a core differentiator.

Bottom dock:
Can remain visible with Rhymes selected if playing a rhyme.
If playing a story, Stories selected.
For prototype, use Rhymes selected.

Top:
- Back arrow
- Small badge: “Now Playing”
- Title: “Twinkle Twinkle Little Star”
- Right-side icon/button:
  - Lyrics On
  - book/lyrics icon

Main hero scene:
- Large rounded scene image area occupying upper half.
- Use local scene image based on current timestamp.
- Show badge:
  “Scene 2 / 5”
- Example scene:
  Night sky, smiling star, moon, clouds, cottage, sparkles.

Scene behavior:
Use hardcoded scene timeline for prototype.

Example:
[
  { scene: 1, start: 0, end: 15, image: player_scene_1.png },
  { scene: 2, start: 15, end: 35, image: player_scene_2.png },
  { scene: 3, start: 35, end: 55, image: player_scene_3.png },
  { scene: 4, start: 55, end: 75, image: player_scene_4.png },
  { scene: 5, start: 75, end: 105, image: player_scene_5.png }
]

When current audio time reaches new scene:
- Update background scene image
- Highlight selected scene thumbnail
- Use soft fade transition if possible

Transcript/read-along panel:
- Rounded cream card overlapping bottom of scene
- Label: “Read Along”
- Show max 2 lines at a time

Example lines:
Line 1: “Twinkle twinkle little star”
Line 2: “How I wonder what you are”

Word highlighting behavior:
Use hardcoded word timings for prototype.

Visual states:
- Past words: dim purple
- Current word: bright pink/yellow capsule highlight
- Upcoming words: dark readable text

Example:
“Twinkle twinkle little star”
If current word is “little”:
- “Twinkle twinkle” = dim purple
- “little” = pink rounded highlight with white text
- “star” = dark text

Controls:
- Progress bar
- Elapsed time: 00:18
- Total time: 01:45
- Star-shaped slider handle if possible
- Replay 10s button
- Previous button
- Large central Play/Pause button
- Next button
- Volume button

Scene thumbnail strip:
- Show 5 small thumbnails
- Label them 1–5
- Highlight current scene
- Tapping thumbnail jumps to scene start time

Interactions:
- Play/pause local audio
- Progress updates with audio currentTime
- Scene changes based on time
- Word highlighting updates based on time
- Replay 10s subtracts 10 seconds
- Previous/next moves to previous/next scene or track
- Lyrics On button toggles transcript panel visible/hidden

==================================================

Playlist constraints:
- Max playlists: 10
- Max items per playlist: 50
- Display count as current / 50

- Cannot edit

==================================================
21. RESPONSIVENESS
==================================================

Primary target:
Mobile portrait.

Prototype should work best around:
- 390 x 844
- 393 x 852
- 414 x 896
- 430 x 932

It can also be previewed on desktop but should remain mobile-first.

Use:
- fixed bottom dock
- scrollable content area
- safe padding above/below dock
- no horizontal page overflow except category icon rows


==================================================
22. ACCESSIBILITY / READABILITY
==================================================

Important:
- Use high contrast text
- Avoid tiny labels
- Use large controls
- Keep max 2 transcript lines in player
- Avoid text over busy image unless using a semi-opaque panel
- Buttons should have clear labels
- Use icons plus labels for child comprehension

Player transcript:
- Current word must be highly visible
- Past/upcoming words must remain readable
- Use large font size


==================================================
23. PROTOTYPE SUCCESS CRITERIA
==================================================

The prototype is successful if:
1. Screens visually match the supplied screenshots in spirit and style.
2. Navigation with the five-tab dock works.
3. Rhymes and Stories have round horizontal category icons.
4. Content list appears when category is selected.
5. Kid Mode hides editing/export controls.
6. Parent PIN unlocks Parent Mode.
7. Parent Mode playlist editor shows add/rename/remove/reorder/export controls.
8. Player screen shows scene image, transcript, word highlighting, progress, and controls.
9. Register Toy and Verify OTP screens are present and clickable.
10. All data and media are local/static only.


==================================================
24. DO NOT BUILD YET
==================================================

Do not build:
- real backend integration
- real OTP
- real login/signup
- real SD card export
- real downloads
- real background sync
- real security layer
- real device ownership
- server transcript generation
- server scene generation

This is only an initial UX prototype/wireframe with local mock files.
